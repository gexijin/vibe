# Claude Code: 基本操作

AIアシスタントを使ったコーディングは、最初は圧倒されるように感じるかもしれません。Claude Codeは、隣に座っている知識豊富な同僚のようなものです。やりたいことを平易な日本語で説明すれば、目標達成を手伝ってくれます。このチュートリアルでは、会話の開始からワークスペースの管理まで、日々使用する基本的な操作を学びます。

## 主要な概念

- **Claude Code** - コードベースを理解し、会話を通じてコードの記述、デバッグ、理解を支援するAI搭載のコマンドラインツール
- **REPL（Read-Eval-Print Loop）** - コマンドを入力し、Claudeが応答し、終了するまで会話が続く対話型セッション
- **コンテキスト** - Claudeが記憶するコードと会話履歴の量。時間とともに満杯になる作業メモリのようなもの
- **スラッシュコマンド** - `/`で始まる組み込みショートカットで、履歴のクリアやヘルプの表示など特定のアクションを実行

## 必要なもの

- [Claude Codeがインストール済み](https://code.claude.com/docs/en/quickstart)で、Claude Pro/MaxサブスクリプションまたはAPIキーが有効
- ターミナルまたはコマンドプロンプトの基本的な使い方の知識
- インターネット接続
- 15〜20分

## ステップ1: ターミナルを開く

- **Windows**: Windowsキーを押し、`Terminal`または`PowerShell`と入力してEnterを押します
- **Mac**: `Cmd+Space`を押し、`Terminal`と入力してEnterを押します
- **Linux**: `Ctrl+Alt+T`を押すか、アプリケーションメニューからTerminalを見つけます

コマンドを入力できるテキストウィンドウが開きます。

## ステップ2: デモプロジェクトを取得する

実際のデータサイエンスプロジェクトを使用して、Claude Codeの機能を探索します。Gitでクローンするか、直接ダウンロードできます。

**オプションA: Gitでクローン（Gitがインストールされている場合）:**

```
git clone https://github.com/gexijin/data_projects
cd data_projects
```

**オプションB: Gitなしでダウンロード:**

- Webブラウザで[https://github.com/gexijin/data_projects](https://github.com/gexijin/data_projects)にアクセスします
- 右上近くの緑色の**Code**ボタンをクリックします
- **Download ZIP**をクリックします
- ZIPファイルを覚えやすい場所（デスクトップやドキュメントフォルダなど）に解凍します
- ターミナルで、解凍したフォルダに移動します：
  - **Windows**: `cd C:\Users\YourName\Downloads\data_projects-main`
  - **Mac/Linux**: `cd ~/Downloads/data_projects-main`

`YourName`を実際のユーザー名に置き換え、別の場所に解凍した場合はパスを調整してください。

## ステップ3: フォルダからClaude Codeを起動する

ターミナルで（data-projectsフォルダ内にいることを確認してください）、次のように入力します：

```bash
claude
```

ウェルカムメッセージとClaude Codeのプロンプトが表示されます。

## ステップ4: プロジェクトについて質問する

Claude Codeは必要に応じて自動的にファイルを読み取ります。プロジェクトを理解するために、以下の質問を試してみてください：

**フォルダ構造について質問:**

```
このプロジェクトのフォルダ構造は何ですか？
```

**テクノロジーについて質問:**

```
このプロジェクトはどのようなテクノロジーとライブラリを使用していますか？
```

**最近の変更について質問:**

```
このプロジェクトに対して最後に行われた変更は何ですか？
```

Claudeは（利用可能な場合）Git履歴をチェックし、最近のコミットについて教えてくれます。

コードについて何でも自然言語で質問できます。必要に応じてファイルを読み取って質問に答えます。

## ステップ5: 重要なスラッシュコマンド

`/`と入力してEnterを押すと、利用可能なすべてのコマンドが表示されます。最も重要なものは以下の通りです：

**すべてのコマンドを表示:**

```
/
```

これにより、すべてのスラッシュコマンドのメニューが表示されます。矢印キーで閲覧し、Enterで選択、Escでキャンセルします。

**ヘルプを取得:**

```
/help
```
Claude Codeの使用に関するドキュメントを表示します。



**コンテキスト使用状況を確認:**
Claudeの「作業メモリ」であるコンテキストを管理することが重要です。

```
/context
```

**会話履歴をクリア:**

コンテキストが満杯になったら、`/clear`で新しい会話を開始します。
```
/clear
```
現在の会話を消去し、新たに開始します。トピックを変更したい場合や、会話が長くなりすぎた場合に使用します。コンテキストを管理することは不可欠です。


**Claude Codeを終了:**

```
/exit
```

セッションを終了し、通常のターミナルプロンプトに戻ります。**Ctrl + C**を2回押すこともできます。

## ステップ6: キーボードショートカット

これらのショートカットにより、Claude Codeでの作業が速くなります：

- **Shift+Tab** - プラン、編集、または通常モードを切り替え - 複雑なタスクには最初にプランを立てる
- **Alt+Enter**（Windows/Linux）または**Option+Return**（Mac） - メッセージを送信せずに新しい行を追加
- **Ctrl+C** - 現在の操作またはClaudeの応答をキャンセル
- **Ctrl+D** - Claudeが許可を求めたときにファイルの変更を承認
- **Esc** - メニューを閉じるか、現在の入力をキャンセル

## ステップ7: 常にCLAUDE.mdファイルを作成する

CLAUDE.mdファイルは、プロジェクトのClaudeに対する取扱説明書です。セッション間で保持されるため、Claudeはプロジェクトに関する重要なコンテキストを記憶します。

**ファイルを作成:**

```
/init
```

Claudeはプロジェクトの概要を含むファイルを作成します。このファイルはプロジェクトルートに残り、新しいセッションを開始するたびにClaudeが自動的に読み取ります。

プロジェクト固有の指示、コーディング規約、ファイルの目的などの重要なコンテキストを追加するために、プロジェクトフォルダ内でいつでもCLAUDE.mdを編集できます。

## ステップ8: ファイルやコード行を参照する

`@`を使用して特定のファイルを参照できます：

```
@Visualization/Matplotlib/Nested_Pie_Chart.ipynb のコードを説明してください
```

Claudeはノートブックを読み取り、それが何をするか、どのように機能するか、コードが何を達成するかを説明します。これにより、ファイルをコンテキストに効果的に追加できます。

VS CodeからClaude Codeを使用し、Claude Code拡張機能がインストールされている場合は、単にファイルを開くだけでコンテキストに追加できます。コマンドウィンドウの右下に`In Nested_Pie_Chart.ipynb`と表示されます。そうすれば、Claudeはあなたがこのファイルについて話していることを理解します。

さらに、数行のコードを選択すると、Claudeは**3 lines selected**と表示します。これらの行に素早く変更を加えたり、質問したりできます。したがって、VS CodeからClaude Codeを使用することを強くお勧めします。


## ステップ9: Linuxコマンドでアクションを実行する

Claudeは様々な形式でLinuxコマンドを実行してアクションを実行できます。

- ソフトウェアをインストール
  ```
  pandasライブラリをインストール
  ```

- バージョン管理を開始
  ```
  Gitを使用して変更の追跡を開始します。私の名前はジェームズ・ボンドで、メールアドレスはbond@earth.comです
  ```
- 変更をコミット
  ```
  これらの変更をコミット
  ```
- データを検索してダウンロード
  ```
  UCIからワイン品質データセットをダウンロード。wineという新しいフォルダに入れてください
  ```

- コードを実行
  ```
  ネストされた円グラフのコードを通常のPythonスクリプトとして書き直してください。
  実行して、新しいコードとプロットを同じフォルダに保存してください。
  ```

コードの説明を求めたばかりなので、この曖昧な質問をすることができます。この後、多くのことが起こります。Claudeはソフトウェアをインストールし、エラーのトラブルシューティングを行い、環境の問題を解決します - すべて独自に。



基本的に、Linux bashコマンドの専門家があなたの自由に使えます。権限を管理し、アクションを承認する限り、非常に生産的になれます。

## トラブルシューティング

- **「Command not found」エラー** - Claude CodeがインストールされていないかPATHに含まれていません。`npm install -g @anthropic-ai/claude-code`を実行してインストールしてください。
- **Claudeが古い情報を提供する** - `/clear`でコンテキストをクリアして、再度質問してください。長い会話はClaudeのメモリを満杯にする可能性があります。
- **ファイル変更が機能しない** - プロジェクトフォルダに書き込み権限があることを確認してください。Claudeはファイルを変更する前に承認を求めます - Ctrl+Dを押して承認してください。
- **コンテキストが急速に満杯になる** - `/context`を使用して使用状況を確認してください。ほぼ満杯になったら、`/clear`を使用してクリーンな状態で新しい会話を開始してください。

## ワークフロー概要

Claude Codeでの作業は、このパターンに従います：

- プロジェクトフォルダで`claude`を使用してClaudeを起動
- 自然言語で質問
- Claudeは必要に応じてファイルを読み取り、応答
- コード変更をリクエスト - Claudeはファイルを変更する前に許可を求めます
- トピックを変更する場合やコンテキストが満杯になった場合は`/clear`を使用
- 完了したら`/exit`を使用
