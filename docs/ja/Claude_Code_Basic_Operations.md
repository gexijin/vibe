# Claude Code: 基本操作

AIアシスタントを使ったコーディングは、最初は圧倒されるように感じるかもしれません。Claude Codeは、隣に座っている知識豊富な同僚のようなものです。やりたいことを平易な日本語で説明すれば、目標達成を手伝ってくれます。このチュートリアルでは、会話の開始からワークスペースの管理まで、日々使用する基本的な操作を学びます。

## 主要な概念

- **ワークスペース** - フォルダからClaude Codeを起動すると、そのフォルダがセッションのワークスペースになります
- **REPL（Read-Eval-Print Loop）** - コマンドを入力し、Claudeが応答し、終了するまで会話が続く対話型セッション
- **コンテキスト** - Claudeが記憶するコードと会話履歴の量。時間とともに満杯になる作業メモリのようなもの
- **スラッシュコマンド** - `/`で始まる組み込みショートカットで、履歴のクリアやヘルプの表示など特定のアクションを実行

## 必要なもの

- [Windows](Install_CLAUDE_Code_Win.md)または[Mac](Install_Claude_Code_MacOS.md)のチュートリアルからClaude Codeをインストール済み
- Windowsを使用している場合はWSLがインストール済み
- ターミナルまたはコマンドプロンプトの基本的な使い方の知識
- 15〜20分

## ステップ1: ターミナルを開く

- **Windows**: Windowsキーを押し、`Ubuntu`と入力してEnterを押し、WSLターミナルを開きます
- **Mac**: `Cmd+Space`を押し、`Terminal`と入力してEnterを押します

コマンドを入力できるウィンドウが開きます。

## ステップ2: デモプロジェクトファイルを取得する

実際のデータサイエンスプロジェクトを使用して、Claude Codeの機能を探索します。Gitでクローンするか、直接ダウンロードできます。

**オプションA: Gitでクローン（Gitがインストールされている場合）:**

```bash
git clone https://github.com/gexijin/data_projects
cd data_projects
```

**オプションB: Gitなしでダウンロード:**

- Webブラウザで[https://github.com/gexijin/data_projects](https://github.com/gexijin/data_projects)にアクセスします
- 右上近くの緑色の**Code**ボタンをクリックします
- **Download ZIP**をクリックします
- ZIPファイルをダウンロードフォルダの**data_projects**というフォルダに解凍します
- ターミナルで、解凍したフォルダに移動します：
  - **Windows**: `cd /mnt/c/Users/YourName/Downloads/data_projects`
  - **Mac/Linux**: `cd ~/Downloads/data_projects`

`YourName`を実際のユーザー名に置き換え、別の場所に解凍した場合はパスを調整してください。Windowsユーザーの場合、WindowsファイルはLinuxシステムから/mnt/cでアクセスできます。

## ステップ3: フォルダからClaude Codeを起動する

ターミナルで（data_projectsフォルダ内にいることを確認してください）、次のように入力します：

```bash
claude
```

[インストールチュートリアル](Install_CLAUDE_Code_Win.md)に従って、Claudeサブスクリプションでログインしてください。ログイン後、ウェルカムメッセージとClaude Codeのプロンプトが表示されます。

正常にログインすると、ウェルカムメッセージとClaude Codeのプロンプトが表示されます。

## ステップ4: プロジェクトについて質問する

Claude Codeは必要に応じて自動的にファイルを読み取ります。プロジェクトを理解するために、以下の質問を試してみてください：

**フォルダ構造について質問:**

```
このプロジェクトのフォルダ構造を教えてください
```

**テクノロジーについて質問:**

```
このプロジェクトで使用しているテクノロジーとライブラリを説明してください
```

**最近の変更について質問:**

```
このプロジェクトへの最新の変更内容は何ですか？
```

Claudeは（利用可能な場合）Git履歴をチェックし、最近のコミットについて教えてくれます。

コードについて何でも自然言語で質問できます。Claudeは必要に応じてファイルを読み取って質問に答えます。

## ステップ5: 重要なスラッシュコマンド

`/`と入力してEnterを押すと、利用可能なすべてのコマンドが表示されます。最も重要なものは以下の通りです：

**すべてのコマンドを表示:**

```
/
```

これにより、すべてのスラッシュコマンドのメニューが表示されます。矢印キーで閲覧し、Enterで選択、Escでキャンセルします。

**ヘルプを取得:**

```
/help
```
Claude Codeの使用に関するドキュメントを表示します。



**コンテキスト使用状況を確認:**
Claudeの「作業メモリ」であるコンテキストを管理することが重要です。

```
/context
```

**会話履歴をクリア:**

コンテキストが満杯になったり、トピックを変更したい場合は、`/clear`で新しい会話を開始します。
```
/clear
```
現在の会話を消去し、新たに開始します。会話が長くなりすぎた場合にも使用します。コンテキストを管理することは重要です。


**Claude Codeを終了:**

```
/exit
```

セッションを終了し、通常のターミナルプロンプトに戻ります。**Ctrl + C**を2回押すこともできます。

## ステップ6: キーボードショートカット

これらのショートカットにより、Claude Codeでの作業が速くなります：

- **Shift+Tab** - プラン、編集、または通常モードを切り替え - 複雑なタスクには最初にプランを立てる
- **Alt+Enter**（Windows/Linux）または**Option+Return**（Mac） - メッセージを送信せずに新しい行を追加
- **Ctrl+C** - 現在の操作またはClaudeの応答をキャンセル
- **Ctrl+D** - Claudeが許可を求めたときにファイルの変更を承認
- **Esc** - メニューを閉じるか、現在の入力をキャンセル

## ステップ7: CLAUDE.mdファイルを作成する

CLAUDE.mdファイルは、プロジェクトのClaudeに対する取扱説明書です。セッション間で保持されるため、Claudeはプロジェクトに関する重要なコンテキストを記憶します。

**ファイルを作成:**

```
/init
```

Claudeはプロジェクトの概要を含むファイルを作成します。このファイルはプロジェクトルートに保存され、新しいセッションを開始するたびにClaudeが自動的に読み取ります。

プロジェクト固有の指示、コーディング規約、ファイルの目的などの重要なコンテキストを追加するために、いつでもCLAUDE.mdを編集できます。

## ステップ8: ファイルやコード行を参照する

`@`を使用して特定のファイルを参照できます：

```
@Visualization/Matplotlib/Nested_Pie_Chart.ipynb のコードを説明してください
```

Claudeはノートブックを読み取り、その内容、動作の仕組み、コードが何を達成するかを説明します。これにより、ファイルをコンテキストに効果的に追加できます。

VS CodeでClaude Code拡張機能を使用している場合は、ファイルを開くだけでコンテキストに追加されます。コマンドウィンドウの右下に`In Nested_Pie_Chart.ipynb`と表示されます。これで、Claudeはあなたがこのファイルについて話していることを理解します。

さらに、数行のコードを選択すると、Claudeは**3 lines selected**と表示します。これらの行に素早く変更を加えたり、質問したりできます。このため、VS CodeからClaude Codeを使用することを強くお勧めします。


## ステップ9: Linuxコマンドでアクションを実行する

Claudeは自然言語の指示をLinuxコマンドに変換して実行できます。

- ソフトウェアをインストール
  ```
  pandasライブラリをインストールしてください
  ```

- バージョン管理を開始
  ```
  Gitを使用して変更の追跡を開始してください。私の名前はジェームズ・ボンドで、メールアドレスはbond@earth.comです
  ```
- 変更をコミット
  ```
  これらの変更をコミットしてください
  ```
- データを検索してダウンロード
  ```
  UCIからワイン品質データセットをダウンロードして、wineという新しいフォルダに入れてください
  ```

- コードを実行
  ```
  ネストされた円グラフのコードを通常のPythonスクリプトとして書き直してください。実行して、新しいコードとプロットを同じフォルダに保存してください
  ```

コードの説明を求めた直後なので、このような曖昧な指示でも理解できます。Claudeはソフトウェアのインストール、エラーのトラブルシューティング、環境の問題解決をすべて自動で行います。

基本的に、Linux bashコマンドの専門家があなたのために働いてくれます。権限を管理し、アクションを承認する限り、非常に生産的に作業できます。

## トラブルシューティング

- **「Command not found」エラー** - Claude CodeがインストールされていないかPATHに含まれていません。`npm install -g @anthropic-ai/claude-code`を実行してインストールしてください。
- **Claudeが古い情報を提供する** - `/clear`でコンテキストをクリアして、再度質問してください。長い会話はClaudeのメモリを満杯にする可能性があります。
- **ファイル変更が機能しない** - プロジェクトフォルダに書き込み権限があることを確認してください。Claudeはファイルを変更する前に承認を求めます。Ctrl+Dを押して承認してください。
- **コンテキストが急速に満杯になる** - `/context`を使用して使用状況を確認してください。ほぼ満杯になったら、`/clear`を使用して新しい会話を開始してください。

## ワークフロー概要

Claude Codeでの作業は、次のパターンに従います：

1. プロジェクトフォルダで`claude`を使用してClaudeを起動
2. 自然言語で質問やリクエストを入力
3. Claudeは必要に応じてファイルを読み取り、応答
4. コード変更をリクエスト（Claudeはファイルを変更する前に許可を求めます）
5. トピックを変更する場合やコンテキストが満杯になった場合は`/clear`を使用
6. 完了したら`/exit`を使用
