# 我如何用可复用提示词自动化文档创建

2025年12月，我着手为Claude Code创建一个全面的教程库。目标是：实践性强、重点突出的教程，一步步引导完全的初学者完成实际任务。不堆砌理论。不写大而全的参考手册。只是"做这个，然后这个，再然后这个"，直到你完成一些具体的事情。

从几个文档文件开始，很快发展成为一个拥有100多个教程文档的多语言学习平台。秘诀是什么？我没有全部手写。相反，我构建了一套自定义斜杠命令（本质上是可复用的提示词），将Claude Code变成了一个自动化文档工厂——始终以我实践性强、步骤清晰的风格生成教程。

## 挑战

创建教程非常耗时。每个教程都需要研究当前最佳实践、结构化的初学者友好写作、一致的格式、技术准确性验证、多语言翻译，以及随着工具演进而进行的持续维护。手动为20多个教程做3种以上语言的版本需要数周时间。

## 解决方案：从手动流程到自动化斜杠命令

关键洞察：**自动化经过验证的工作流程，而不是理论上的。**

我没有从头开始设计斜杠命令。相反，我首先通过与Claude的交互式提示手动创建了最初几个教程，通过迭代改进流程，然后让Claude将有效的流程编纂成斜杠命令。

这正是我在[研究论文教程](./Writing_Research_Paper_Claude_Code)中描述的模式：手动完成工作流程，然后在最后一步，让Claude将其保存为可复用的命令。

结果是五个专门的斜杠命令，处理整个教程生命周期。

### [`/tutorial`](../assets/commands/tutorial.md) - 教程生成器

这个命令诞生于交互式创建多个教程的过程，捕获了经过验证的工作流程：

1. **研究**：Claude在网上搜索最新信息——没有过时的版本号或废弃的方法
2. **计划**：Claude展示它学到的内容，推荐方法，概述主要步骤
3. **迭代**：我审查计划，多次提示Claude修改直到满意
4. **写作**：批准后，Claude按照严格的格式规则写作：
   - 顶部的主页链接
   - 吸引人的开头
   - 关键概念
   - 带有动作动词的步骤说明
   - 故障排除部分
   - 创建日期
5. **测试**：我在单独的终端中自己测试步骤，需要时修改
6. **润色**：有时我手动编辑，有时让Claude来做

这个命令在所有教程中强制执行一致的结构。每个教程都感觉像是来自同一个作者——因为它们遵循相同的系统化流程。我生成了20多个教程，涵盖从基本Git操作到高级Docker工作流程的主题。

### [`/review-tutorial`](../assets/commands/review-tutorial.md) - 质量控制机器人

值得注意的是：我只是简单地让Claude"创建一个审查教程的斜杠命令"。没有详细规格。Claude生成了一个全面的三阶段工作流程，包含30多个质量标准，组织成不同类别：

- **内容质量**：标题、开头、关键概念、先决条件、步骤流程、后续步骤、故障排除
- **格式标准**：主页链接、步骤标题、项目符号、加粗/反引号、代码块、段落
- **初学者友好性**：基于菜单的说明、平台差异、精确的点击目标、无术语
- **技术准确性**：正确的命令、当前的UI、现实的时间估计
- **写作质量**：语法、一致的术语、简洁的语言

该命令以结构化报告呈现发现，然后在批准后应用修复。为什么需要这个？两个原因：Claude并不总是严格遵循`/tutorial`规则，而且`/tutorial`命令本身随着我构建更多教程而演进。审查命令让我可以批量润色早期教程，使其符合最新标准。

### [`/translate-chinese`](../assets/commands/translate-chinese.md) & [`/translate-spanish`](../assets/commands/translate-spanish.md) - 本地化引擎

有趣的是，日语翻译是最先完成的——而且没有斜杠命令。我只是在一个提示中让Claude Code将所有教程翻译成日语。Claude自动生成了8个并行运行的子代理，每个同时处理不同的教程。结果非常好，这给了我信心，将流程正式化为中文和西班牙语的斜杠命令。

同样，我只是简单地让Claude"创建一个将教程翻译成中文的斜杠命令"——没有具体指南。Claude生成了一个包含全面规则的六阶段工作流程：

- **翻译规则**：保持代码块、技术术语（Git、Docker、VS Code）、文件路径和URL为英文；翻译说明文本、标题和解释
- **语言指南**：语气和风格规则、常见技术翻译（Click = 点击、Install = 安装）、正式称呼惯例
- **格式要求**：本地化的主页链接、保留的结构、正确的标点符号（中文用。，！？，西班牙语用¿? ¡!）
- **质量审查**：自然流畅检查表、一致的术语、正确的字符使用

斜杠命令准备好后，我让Claude Code使用子代理翻译所有25个教程。整个翻译——两种语言50个新文件——只用了15分钟。

结果：中文、西班牙语和日语目录中共81个翻译教程文件——全部保持一致的质量和结构。

### [`/review-translation`](../assets/commands/review-translation.md) - 翻译维护工具

教程在演进。命令在变化。新部分在添加。这个命令通过四阶段工作流程保持翻译同步：

1. **读取两个版本**：并排加载英文原版和翻译
2. **比较和更新**：识别缺失的部分、过时的内容、更改的命令或URL
3. **质量审查**：检查翻译准确性、语言质量、格式一致性
4. **报告和修复**：呈现发现、获得批准、应用更新

质量审查非常彻底——对于日语，它检查自然的措辞（不是逐字翻译）、适当的礼貌程度（です/ます形式）、正确的助词使用（は/が、を、に、で），以及没有不自然的片假名过度使用。当我更新英文教程时，我可以快速将更改传播到所有翻译，同时保持语言质量。

## 使用子代理扩展

对于真正的并行工作，我使用了Claude的子代理功能。在润色日语翻译时，我同时启动多个审查代理，通过协调改进处理19个文件。

斜杠命令用于结构化工作流程，子代理用于并行化，两者的结合创建了一个文档管道，其规模远超手动努力所能达到的。

## 结果

在不到两周的时间里，我创建了20多个涵盖初学者到中级主题的英文教程、3种语言81个翻译教程、通过自动审查流程保证一致的质量，以及带有翻译同步工具的可维护文档。

所有教程遵循相同的结构、写作风格和格式惯例——因为它们是由相同的系统化流程生成的。

## 关键经验

- **先手动，再自动化。** 先手动完成任务，改进流程，然后让Claude将其保存为命令。
- **将命令构建为多阶段工作流程。** 告诉Claude如何思考问题，而不仅仅是做什么。
- **将质量控制纳入流程。** 在创建生成命令的同时创建审查命令。
- **使命令协作而非自主。** 包含批准步骤。Claude处理繁琐的部分；你做战略决策。

## 结论

构建这个教程库教会我，自动化不是要把人从流程中移除——而是要放大人的判断。我没有手动写100多个教程，但每一个都反映了我的标准、我的结构和我的批准。

斜杠命令将Claude Code从一个有用的助手转变为一个文档工厂，按我的指示工作，维护我的标准，并扩展到我需要的任何项目规模。

如果你有重复的文档任务，不要继续手动做。一次构建斜杠命令，然后部署它几十次或几百次。

这就是系统化自动化的力量。
