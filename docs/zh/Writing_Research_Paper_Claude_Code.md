# 使用 Claude Code 撰写研究论文

使用 Claude Code 作为助手进行研究、头脑风暴、规划、起草和编辑来撰写研究论文。然后通过创建模板斜杠命令来自动化此流程。查看使用此工作流程创建的[示例论文](./example_paper.md)。

## 核心概念

- **工作流程** - 研究 → 头脑风暴 → 聚焦研究 → 大纲 → 起草 → 手动修订 → AI 润色 → 添加摘要 → 改进标题 → 检查参考文献 → AI 校对
- **AI 助手** - 人类做出关键决策,AI 完成繁琐的工作
- **Claude Code** - 智能代理 AI 系统,通过简单的请求搜索网络、组织研究并起草内容
- **斜杠命令** - 可在 Claude Code 中重复使用的自定义详细提示

## 准备工作

- 完成 [Windows 上 VS Code 中的 Claude Code](./Claude_Code_in_VS_Code_Win) 或 [Mac 上 VS Code 中的 Claude Code](./Claude_Code_in_VS_Code_Mac)
- 30-40 分钟

## 步骤 1:创建项目文件夹

为你的研究论文创建一个文件夹:

- 打开**文件资源管理器**(Windows) 或 **Finder**(Mac)
- 导航到**文档**
- 创建一个名为 `test_claude` 的新文件夹

此项目的所有内容都在此文件夹中进行。

## 步骤 2:启动 VS Code
Windows 系统:
- 点击**Windows 开始按钮**(屏幕左下角)
- 在搜索框中输入 `Visual Studio Code` 或 `VS Code`
- 当搜索结果中出现 **Visual Studio Code** 时点击它
- VS Code 打开并显示欢迎标签页 - 你可以关闭此标签页
- 查看 VS Code 左下角 - 你会看到一个蓝色或绿色图标
- 点击此图标打开远程连接菜单
- 从菜单中选择 **Connect to WSL**
- VS Code 将重新加载并连接到你的 Ubuntu 安装
- 左下角现在应该显示 **WSL: Ubuntu**

Mac 系统:
- 打开 **Finder** 并转到**应用程序**
- 找到 **Visual Studio Code** 并双击它
- 如果看到警告"Visual Studio Code 是从互联网下载的应用程序",请点击**打开**
- VS Code 打开并显示欢迎标签页 - 你可以关闭此标签页

## 步骤 3:在 VS Code 中打开文件夹
通过 WSL 的 Windows 系统:
- 在 VS Code 中(仍然连接到 WSL),点击菜单栏中的**文件**,然后点击**打开文件夹**
- 顶部中央会出现一个**打开文件夹**下拉菜单。
- 通过输入以下内容找到你的文件夹:
  ```
  /mnt/c/Users/YOUR_USERNAME/Documents/test_claude
  ```
  将 `YOUR_USERNAME` 替换为你的 Windows 用户名(如 `John.Smith`)
- 点击 **OK**。VS Code 将重新加载你的 `test_claude` 文件夹


Mac 系统:
- 在 VS Code 中,点击菜单栏中的**文件**,然后点击**打开文件夹**
- 导航到并选择 `test_claude` 文件夹
- 点击**打开**(Mac) 或 **OK**(Windows)
- 如果提示"你信任作者吗?",请点击**是,我信任作者**

## 步骤 4:启动 Claude Code

- 打开终端:点击 VS Code 主菜单中的**终端**,然后点击**新建终端**
- 在终端面板中,输入:
  ```
  claude
  ```

按照安装教程登录你的 Claude 订阅([Windows](Install_CLAUDE_Code_Win.md) | [Mac](Install_Claude_Code_MacOS.md))。登录后,你将看到欢迎消息和 Claude Code 提示符。

## 步骤 5:初步研究

让 Claude 研究你的主题。将此提示复制并粘贴到 Claude Code 中,将主题替换为你自己的:

```
我正在撰写一篇约 1,000 字的关于职场 AI 采用的研究论文。
搜索最近的数据(2023-2025)关于:
- AI 工具带来的生产力提升
- 工作岗位替代的担忧和统计数据
- 来自公司的真实案例研究

要求:
- 优先考虑同行评审的研究和可信的行业报告
- 避免轶事和观点文章
- 列出每个来源并附上 1-3 句话的摘要
- 按主题对来源进行分组

保存为 general_research.md
```

Claude 搜索网络并将研究结果组织到名为 `general_research.md` 的文件中。这可能需要一两分钟。完成后,请求摘要:

```
给我一个简要总结。
```

查看 Claude 的摘要以快速了解研究概况。

## 步骤 6:阅读来源
阅读研究文档并点击原始来源以验证信息:

1. 点击左侧资源管理器面板中的 `general_research.md`
2. 预览格式化的文档:右键点击 `general_research.md` 标签页并选择**打开预览**
3. 点击来源链接阅读原始文章和研究

## 步骤 7:头脑风暴你的角度

让 Claude 帮助头脑风暴:

```
根据这项研究,建议我可以为这篇论文采取的 3-4 个角度。
```

查看这些角度并选择最让你感兴趣的那个。

## 步骤 8:聚焦研究

现在你有了自己的角度,请让 Claude 进行针对性研究:

```
我想专注于[你选择的角度]。搜索更具体的数据
和支持这一观点的示例。保存为 focused_research.md
```

Claude 找到针对性信息。

## 步骤 9:创建你的计划

让 Claude 根据你选择的角度创建大纲:

```
根据聚焦研究和我选择的角度,给我 3 个简要大纲选项。
使用项目符号标记叙述流程。
保存为 outlines.md
```

查看大纲并根据需要让 Claude 进行调整(如"让第 2 部分更多地关注案例研究"或"添加关于局限性的部分")。

## 步骤 10:起草论文

选择你的大纲并让 Claude 撰写完整草稿:

```
我喜欢大纲 #2 [你选择的选项]。

根据大纲和研究撰写一篇约 1,000 字的研究论文。

结构:
- 引言:吸引人的开头、背景、论点陈述
- 正文:2-3 个部分,包含论证、统计数据和示例
- 结论:总结和影响

风格:
- 清晰、简洁的句子(平均 15-20 个字)
- 主动语态,分析性语气
- 将统计数据融入散文中(不要使用项目符号列表)
- 段落之间过渡流畅

引用:
- 在声明后使用编号引用 [1]、[2] 等
- 包含 5-15 个参考文献
- 在末尾添加参考文献部分

保存为 paper.md
```

Claude 撰写草稿。

## 步骤 11:手动修订

在文本编辑器中打开 `paper.md`。仔细阅读并进行你自己的修订:
- 添加你的个人声音和见解
- 调整论证以匹配你的思维
- 修复任何笨拙的措辞
- 确保引用准确

在编辑器中保存你的更改。

## 步骤 12:AI 润色

让 Claude 改进特定部分:

```
让引言更有吸引力,加入引人入胜的开头。
```

## 步骤 13:添加摘要

让 Claude 在开头添加简短的执行摘要:

```
在论文开头添加摘要。写 2-3 个简短的句子
总结论文内容。
```

Claude 将添加摘要。查看并编辑以确保准确捕捉论文的精髓。

## 步骤 14:改进标题

让 Claude 为你提供标题的几个选项:

```
给我几个标题选项。让它更有吸引力。
```

选择一个标题。添加你自己的特色:

```
我喜欢选项 #2 [你的选择]。编辑文件。
```

## 步骤 15:检查参考文献(可选)
让 Claude 验证你的引用是否一致且完整:
```
检查我论文中的所有参考文献:
- 验证每个引用编号 [1]、[2] 等是否有匹配的参考文献
- 验证列表中的每个参考文献是否在论文中实际被引用
- 检查作者姓名和标题是否一致
- 验证引用的数据和示例是否出现在来源中
```

注意:Claude 只能验证公开访问的来源。对于付费墙后的文章,请手动检查你的引用是否与你阅读的内容匹配。

## 步骤 16:AI 校对

让 Claude 进行最终校对:

```
对论文进行最终校对:
- 修复任何拼写和语法错误
- 确保整体格式一致
- 检查流畅性和过渡
```

查看 Claude 的更改。

**导出到 Word:** 在 VS Code 中,右键点击 `paper.md` 标签页并选择**打开预览**。点击预览窗格内部,然后点击**编辑 > 全选**和**编辑 > 复制**。粘贴到 Microsoft Word 中——格式将被保留。

## 步骤 17:为未来论文创建斜杠命令

将此工作流程保存为可重复使用的斜杠命令,用于你的下一篇研究论文:

```
创建一个名为 /research-paper 的斜杠命令,引导我完成
整个工作流程。保存它,以便我可以将其用于未来关于各种主题的研究论文。
```

Claude 将在你的 `.claude/commands/` 文件夹中创建自定义斜杠命令。它是一个包含提示的 Markdown 文件。你可以通过简单地输入 `/research-paper [你的主题]` 来开始你的下一篇研究论文!

## 步骤 18:获取反馈

LLM 没有记忆。对于每个响应,我们必须发送聊天会话中所有先前的对话以提供上下文。随着交互变长,我们需要注意上下文的长度。如果它不相关,或者我们可以提供更清晰的上下文,我们将从全新开始中受益。

清除 Claude 的记忆以获取对你论文的反馈。输入:

```
/clear
```

这将删除所有对话历史,因此 Claude 将用全新的眼光阅读你的论文。现在请求反馈:

```
阅读 paper.md 并给我诚实的反馈。
```

Claude 将批评你的论文,而不会受到曾经撰写过它的影响。根据反馈进行最终修订。


## 完整工作流程

1. **创建文件夹** - 在文档中设置项目文件夹
2. **启动 VS Code** - 打开 VS Code 并连接到 WSL(Windows)
3. **在 VS Code 中打开** - 在 VS Code 中打开文件夹
4. **启动 Claude Code** - 从终端启动 Claude Code(按照安装教程登录)
5. **研究** - Claude 搜索关于你主题的一般信息
6. **阅读来源** - 在文本编辑器中查看研究
7. **头脑风暴** - 选择你的角度/观点
8. **聚焦研究** - Claude 为你选择的角度查找针对性数据
9. **计划** - Claude 根据你的观点创建大纲选项
10. **起草** - Claude 撰写带有参考文献的完整论文
11. **手动修订** - 你添加自己的声音并进行更改
12. **AI 润色** - Claude 改进特定部分
13. **添加摘要** - Claude 创建执行摘要
14. **改进标题** - Claude 建议标题选项,你进行自定义
15. **检查参考文献** - (可选) Claude 验证所有引用
16. **AI 校对** - Claude 修复错误并改进流畅性
17. **创建斜杠命令** - 保存工作流程以用于未来的论文
18. **获取全新反馈** - 清除记忆以进行公正的批评

## 下一步

现在你有了 `/research-paper` 斜杠命令,尝试撰写更多论文:

- **技术:** `/research-paper "社交媒体对青少年心理健康的影响"`
- **商业:** `/research-paper "远程工作生产力"`
- **科学:** `/research-paper "CRISPR 基因编辑伦理"`
- **格式:** 让 Claude `以 LaTeX 格式给我论文`

斜杠命令将自动引导你完成整个工作流程!切换主题时不要忘记清除记忆。

## 故障排除

- **Claude 搜索结果似乎过时:** 指定"搜索 2024-2025 年关于[主题]的数据"
- **草稿太长/太短:** 告诉 Claude:"使其大约为 1,000 字"
- **找不到文件:** 在终端中,输入 `ls` 列出当前文件夹中的所有文件

## 你可以询问 Claude 什么

- `搜索关于[主题]的更新统计数据` - 查找更新的数据
- `总结本论文中的关键论点` - 获取概述
- `让结论更有说服力` - 改进特定部分
- `以 APA 格式引用所有来源` - 格式化参考文献

Claude 通过自然语言处理研究和写作——无需记忆命令!
