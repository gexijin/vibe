# Documentation des Tutoriels Vibe

Ce dépôt contient des tutoriels complets adaptés aux débutants pour utiliser Claude Code, VS Code et les outils de développement associés. Tous les tutoriels sont conçus pour les débutants complets sans expérience préalable en programmation.

## Compétences Disponibles

Les compétences Claude Code étendent les fonctionnalités avec des capacités spécialisées :

- **docx** - Création, édition et analyse de documents Microsoft Word
- **pdf** - Manipulation de PDF, extraction de texte, remplissage de formulaires et génération
- **pptx** - Création, édition et analyse de présentations PowerPoint
- **xlsx** - Création de feuilles de calcul Excel, analyse, formules et visualisation

### Compétences de Recherche
- **medical-research** - Recherche PubMed et synthèse de la littérature scientifique
- **generating-stock-reports** - Recherche d'entreprises et rapports d'analyse boursière

### Compétences d'Analyse de Données
- **csv-stats** - Statistiques de fichiers CSV et analyse de données

## Commandes Slash

Commandes personnalisées disponibles via `.claude/commands/` :

- `/tutorial [sujet]` - Créez un nouveau tutoriel adapté aux débutants suivant le guide de style
- `/review-tutorial [fichier]` - Révisez et polissez un tutoriel existant pour la clarté, la précision et le formatage
- `/review-translation [fichier]` - Révisez et mettez à jour les traductions pour correspondre aux originaux anglais
- `/bioinfo-newsletter` - Générez une newsletter de recherche en bioinformatique à partir du contenu d'un blog

## Directives de Style de Rédaction

Tous les tutoriels suivent ces principes :

### Public Cible
- Débutants complets sans expérience en programmation
- Utilisateurs non techniques apprenant les outils de développement
- Explications claires sans jargon

### Structure
- Lien `[Accueil](./)` en haut
- Accroche engageante (2-4 phrases)
- Section Concepts Clés (3-4 termes maximum)
- Ce Dont Vous Aurez Besoin (prérequis + estimation du temps)
- Instructions étape par étape avec format `## Étape N : Verbe d'Action + Quoi`
- Section Prochaines Étapes (2-3 suggestions)
- Section Dépannage (2-3 problèmes courants)
- Date de création et auteur en bas

### Formatage
- Utilisez des puces (-) pour les sous-étapes, pas de listes numérotées
- Gras pour les éléments d'interface : **Nom du Bouton**
- Backticks pour les commandes : `commande`
- Blocs de code pour les commandes multi-lignes
- Paragraphes courts (2-3 phrases maximum)
- Instructions basées sur les menus plutôt que sur les raccourcis clavier

### Longueur
- Cible d'environ 400 lignes ou moins
- Concentrez-vous sur les informations essentielles
- Supprimez les explications inutiles
- Gardez un contenu actionnable et concis

## Traductions

Des traductions complètes sont disponibles en trois langues :

### Japonais (`ja/`)
Tous les tutoriels anglais ont des versions japonaises correspondantes avec la même structure de noms de fichiers.

### Chinois (`zh/`)
Tous les tutoriels anglais ont des versions chinoises simplifiées correspondantes avec la même structure de noms de fichiers.

### Espagnol (`es/`)
Tous les tutoriels anglais ont des versions espagnoles correspondantes avec la même structure de noms de fichiers.

Chaque dossier de traduction inclut :
- Tous les guides d'installation
- Tous les guides de configuration de VS Code
- Tous les guides de codage (R et Python)
- Tous les guides Git et GitHub
- Tous les tutoriels avancés
- Projets de recherche et de rédaction

Directives de traduction :
- Les termes techniques (Git, Docker, VS Code) sont conservés en anglais
- Les instructions en ligne de commande ne sont jamais traduites
- Les éléments d'interface correspondent à la localisation le cas échéant
- Langage naturel et fluide approprié à chaque locale
- Terminologie cohérente tout au long

## Contribution

Lors de la création ou de la mise à jour de tutoriels :

1. Utilisez la commande `/tutorial [sujet]` pour créer de nouveaux tutoriels
2. Utilisez la commande `/review-tutorial [fichier]` pour réviser les tutoriels existants
3. Suivez les directives de style de rédaction ci-dessus
4. Gardez les tutoriels ciblés et de moins de 400 lignes
5. Testez toutes les commandes et instructions
6. Incluez un dépannage pour les problèmes courants
7. Ajoutez la date de création et l'attribution de l'auteur
